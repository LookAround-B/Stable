# Contributing to Stable Management System

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a feature branch: `git checkout -b feature/amazing-feature`
4. Make your changes
5. Commit your changes: `git commit -m 'Add amazing feature'`
6. Push to the branch: `git push origin feature/amazing-feature`
7. Open a Pull Request

## Code Style

### JavaScript/Node.js
- Use ES6+ syntax
- Use meaningful variable and function names
- Add comments for complex logic
- Use semicolons
- Use 2-space indentation

### React Components
- Use functional components with hooks
- Use meaningful component names
- Keep components small and focused
- Use prop-types or TypeScript for type safety
- Extract reusable components

### CSS
- Use semantic CSS class names
- Follow BEM naming convention where appropriate
- Use flexbox and grid layouts
- Ensure responsive design
- Minimize CSS duplication

### Database
- Use meaningful table and column names
- Add proper indexes for performance
- Use foreign keys for relationships
- Add timestamps for audit trails
- Use proper data types

## Commit Messages

Follow conventional commits:
- `feat: Add new feature`
- `fix: Fix a bug`
- `docs: Update documentation`
- `style: Code style changes`
- `refactor: Refactor code`
- `test: Add or update tests`
- `chore: Update dependencies or config`

## Pull Request Process

1. Update README.md if adding new features
2. Update API.md for API changes
3. Add tests for new features
4. Ensure all tests pass
5. Ensure code follows style guidelines
6. Request review from team members

## Reporting Issues

When reporting issues, please include:
- Clear description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots if applicable
- Environment details (OS, browser, Node version, etc.)

## Questions?

Feel free to open an issue or contact the development team.
