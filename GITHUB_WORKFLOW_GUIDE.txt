DAILY WORKFLOW - STEP BY STEP
================================================================================

BEFORE STARTING YOUR WORKDAY:

Step 1: Pull latest changes from GitHub
   cd "d:\AI Projects\horsestable"
   git pull origin main

This ensures you have the latest code from your team.

---

AFTER MAKING CHANGES:

Step 2: Check what you changed
   git status

You'll see a list of files you modified (shown in red/green)

Step 3: Stage all your changes
   git add .

Step 4: Create a commit (save your changes with a message)
   git commit -m "Add task filtering for groomers and admins"

Good commit message examples:
- "Fix dashboard greeting to show user name and role"
- "Update Google OAuth button styling"
- "Implement attendance toggle switches"
- "Fix ESLint warnings in AttendancePage"

Bad commit message examples:
- "changes" (too vague)
- "fixed bug" (which bug?)
- "update" (not descriptive)

Step 5: Push your changes to GitHub
   git push

Or explicitly:
   git push origin main

Your changes are now on GitHub!

VERIFY: Go to https://github.com/LookAround-B/Stable to see your changes

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: "fatal: unable to access repository"
SOLUTION: Check internet connection and run:
   git remote -v
   git remote set-url origin https://github.com/LookAround-B/Stable.git

PROBLEM: "Please tell me who you are" error
SOLUTION: Configure git user (do this once):
   git config user.name "SaiAkhil066"
   git config user.email "saiakhil066@gmail.com"

PROBLEM: "Your branch is ahead of 'origin/main' by X commits"
SOLUTION: You need to push:
   git push origin main

PROBLEM: Changes won't push
SOLUTION: Pull first, then push:
   git pull origin main
   git push origin main

PROBLEM: "Rejected (non-fast-forward)"
SOLUTION: Someone else pushed changes. Pull and retry:
   git pull origin main
   git push origin main

BEST PRACTICES
================================================================================

‚úÖ DO:
  - Commit frequently (small, logical changes - not huge chunks)
  - Pull before pushing (get latest code first)
  - Write clear commit messages (describe what you changed)
  - Review your changes before committing (use git diff)
  - Test your changes before committing (don't break the app!)
  - Push at end of day (backup your work)

‚ùå DON'T:
  - Don't commit without testing first
  - Don't push .env or secret files (they're in .gitignore for a reason)
  - Don't commit node_modules (they're huge!)
  - Don't force push to main (unless absolutely necessary)
  - Don't ignore merge conflicts (fix them properly)
  - Don't commit too big (break it into smaller pieces)

FOR YOUR TEAM - SETUP INSTRUCTIONS
================================================================================

When your team members clone the repo:

1. CLONE THE REPO:
   git clone https://github.com/LookAround-B/Stable.git
   cd Stable

2. CREATE ENVIRONMENT FILES:
   cp backend/.env.example backend/.env
   cp frontend/.env.example frontend/.env.local

3. FILL IN CREDENTIALS:
   Edit backend/.env and add:
   - DATABASE_URL (PostgreSQL connection)
   - JWT_SECRET
   - GOOGLE_CLIENT_ID
   - GOOGLE_CLIENT_SECRET
   
   Edit frontend/.env.local and add:
   - REACT_APP_GOOGLE_CLIENT_ID

4. INSTALL DEPENDENCIES:
   cd backend && npm install
   cd ../frontend && npm install

5. SETUP DATABASE:
   cd backend
   npx prisma migrate deploy
   npx prisma db seed

6. START SERVERS:
   Terminal 1 (Backend):
      cd backend && npm run dev
   
   Terminal 2 (Frontend):
      cd frontend && npm start

7. OPEN IN BROWSER:
   http://localhost:3001
   
   Test login: admin@test.com / password123
